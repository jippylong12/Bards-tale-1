---
name: Milestone scoping preference
description: User wants roadmap versions grouped by difficulty/feasibility, not by feature category
type: feedback
---

Group roadmap milestones by feasibility, not by feature:
- v2.x = low-hanging fruit (small byte changes, proven safe)
- v3.x = medium difficulty (needs more thought, might work)
- v4.x = needs human help or may be impossible

**Why:** User doesn't want to hit walls mid-milestone. Better to deliver easy wins first, then attempt harder things with clear expectations.

**How to apply:** When creating roadmaps, assess each feature's technical difficulty against actual codebase constraints (memory, addressing, etc.) before assigning to a version. Be honest about what's hard.
